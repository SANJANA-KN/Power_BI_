# Row-Level Security (RLS) in Power BI

## Introduction
Row-Level Security (RLS) restricts data access based on user roles, ensuring users can only view relevant data.

---

## Why RLS is Important
- Protects sensitive data
- Ensures data privacy
- Provides role-based access
- Used in enterprise systems

---

## Example Use Case

- HR Manager → sees all employees  
- Team Manager → sees only team data  
- Employee → sees only personal data  

---

## Types of RLS

### 1. Static RLS
- Fixed rules
- Defined manually

### 2. Dynamic RLS
- Based on user login
- Uses USERNAME() function

---

## How RLS Works
1. Define roles in Power BI Desktop  
2. Assign filters  
3. Publish to Power BI Service  
4. Assign users to roles  

---

## Example DAX for RLS
```DAX
[Department] = "Sales"
```

---

## Best Practices
- Keep roles simple
- Test before publishing
- Use dynamic RLS where possible
- Validate user access

---

## Key Takeaway
RLS ensures secure and controlled access to sensitive business data in Power BI.
