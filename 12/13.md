# Data Refresh Scheduling in Power BI

## Introduction
Data refresh ensures that Power BI reports always show the latest data from the source system.

---

## Types of Refresh

### 1. Manual Refresh
- User refreshes data manually
- Done in Power BI Desktop or Service

### 2. Scheduled Refresh
- Automatically runs at specific time intervals
- Used in Power BI Service

### 3. Real-Time Refresh
- Data updates instantly
- Used in streaming datasets

---

## Why Refresh is Important
- Keeps data updated
- Ensures accurate reporting
- Supports real-time decision-making

---

## Requirements for Scheduled Refresh
- Gateway setup (for on-premise data)
- Credentials configured
- Power BI Service account

---

## Common Issues
- Gateway failure
- Authentication errors
- Data source timeout
- Large dataset delays

---

## Best Practices
- Schedule refresh during off-peak hours
- Optimize dataset size
- Monitor refresh history
- Use incremental refresh if possible

---

## Key Takeaway
Data refresh keeps dashboards updated and ensures accurate, real-time business insights.
